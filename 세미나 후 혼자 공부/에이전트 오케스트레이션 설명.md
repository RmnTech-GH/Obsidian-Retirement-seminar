# 에이전트 오케스트레이션 (Agent Orchestration)

## 개념

**에이전트 오케스트레이션**은 여러 AI 에이전트를 **지휘자(Orchestrator)**가 조율하여 복잡한 작업을 수행하는 방식입니다.

---

## 구조 다이어그램

```mermaid
flowchart TB
    User["👤 사용자"]
    Orch["🎼 오케스트레이터<br/>(지휘자)"]

    subgraph Agents["AI 에이전트들"]
        A1["🔍 검색 에이전트"]
        A2["💻 코딩 에이전트"]
        A3["📝 문서 에이전트"]
        A4["🧮 분석 에이전트"]
    end

    subgraph Tools["도구들"]
        T1["웹 검색"]
        T2["파일 읽기/쓰기"]
        T3["코드 실행"]
        T4["데이터베이스"]
    end

    User -->|"복잡한 요청"| Orch
    Orch -->|"작업 분배"| A1
    Orch -->|"작업 분배"| A2
    Orch -->|"작업 분배"| A3
    Orch -->|"작업 분배"| A4

    A1 --> T1
    A2 --> T2
    A2 --> T3
    A3 --> T2
    A4 --> T4

    A1 -->|"결과"| Orch
    A2 -->|"결과"| Orch
    A3 -->|"결과"| Orch
    A4 -->|"결과"| Orch

    Orch -->|"최종 답변"| User
```

---

## 작업 흐름

```mermaid
sequenceDiagram
    participant U as 👤 사용자
    participant O as 🎼 오케스트레이터
    participant A as 🤖 에이전트들
    participant T as 🔧 도구들

    U->>O: 복잡한 작업 요청
    O->>O: 작업 분석 & 계획 수립

    loop 각 하위 작업
        O->>A: 작업 할당
        A->>T: 도구 사용
        T-->>A: 결과 반환
        A-->>O: 작업 완료 보고
    end

    O->>O: 결과 통합
    O-->>U: 최종 결과 전달
```

---

## 비유: 오케스트라

| 오케스트라 | 에이전트 오케스트레이션 |
|-----------|----------------------|
| 지휘자 | 오케스트레이터 (메인 AI) |
| 연주자들 | 각 전문 에이전트 |
| 악기 | 도구 (Tools) |
| 악보 | 작업 계획 |
| 공연 | 최종 결과물 |

---

## 장점

1. **병렬 처리** - 여러 에이전트가 동시에 작업
2. **전문화** - 각 에이전트가 특정 영역에 특화
3. **복잡한 작업 처리** - 큰 작업을 작은 단위로 분할
4. **확장성** - 필요에 따라 에이전트 추가 가능
