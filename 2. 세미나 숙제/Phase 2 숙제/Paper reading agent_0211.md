
# Agent화 설계문서

## 0. 업무 분석

### 1. 업무명
- 논문 분석 보고서 작성
### 2. 현재 수행 방식
- 소요 시간: 매주 금요일 60분
- 반복 주기: 매주 1회
- 수작업 내용: 논문 네이밍, 논문 분류, 논문 PDF 파일 markdown으로 변환, 변환된 markdown 파일 분석 요청
### 3. 입력 정보
- 분석하고자 하는 논문
- 용도(단순 내용 파악, 세부 내용 분석)
### 4. 출력 결과물
- 분석 대상 논문 md파일
- 분석 대상 논문 분석 보고서 md파일
### 5. 자동화 가능 부분
- PDF 파일을 md 파일로 변환
- 변환한 md 파일 후처리
- 형식 통일 및 정리
- 분석 결과 도출
### 6. 사람이 해야 할 부분
- 최종 검토 및 수정

## 1. 프로젝트 개요

### 프로젝트명
논문 자동 분석 Agent

### 목표
대상과 목적 입력 정보를 바탕으로 논문을 pdf 파일에서 md파일로 변환 및 분석 보고서 md 파일 자동 생성

### 예상효과
- 기존소요시간 : 30분
- 자동화 후 예상 소요시간 : 5분
- 절감시간 : 25분 / 주
## 2. Agent 설계

### 입력 (Input)
- 분석 대상 논문 (텍스트)
- 분석 용도 (단순 내용 파악, 세부 내용 분석 중 택1)

### 처리과정 (Process)
1. 입력정보 파싱
2. 입력 정보 기판 대상 논문들 pdf에서 md 파일로 변환 및 저장
   (python의 pymupdf411m 패키지 활용, resources 폴더의 pdf_to_md.py 이용)
3. md파일로 변환된 논문 분석 용도에 따라 템플릿 적용
4. 내용, 형식 정리
5. 논문 분석 보고서 md 파일로 저장
### 출력 (Output)
1. 1차 출력 파일
   파일명 : 분석 대상 논문 파일명_YYYYMMDD.md
   형식 : 마크다운
2. 2차 출력파일
   파일명 : 분석 용도_분석 대상 논문 파일명_YYYYMMDD.md
   형식 : 마크다운

## 3. 프롬프트 설계

### Agent 지시 프롬프트

[역할]
당신은 논문 pdf 파일을 md로 변환하는 용도에 맞게 논문을 분석하고 요약 정리하는 AI 비서입니다.

[맥락]
사용자가 입력한 정보를 바탕으로 먼저 논문을 md파일로 변환합니다
다음으로 변환한 논문 md 파일을 이용해서 용도에 맞게 논문을 분석하고 요약 정리합니다.
분석 보고서는 템플릿 양식에 맞춰야 합니다.

[지시]
1. 사용자에게 분석하고자 하는 대상 논문을 물어보세요.
2. 대상 논문의 분석 용도를 물어보세요.
3. 입력받은 정보로 resources 폴더에 있는 "pdf_to_md.py" 파일을 이용해서 대상 논문 pdf 파일을 md 파일로 변환해주세요.
4. 입력받은 분석 용도 정보로 논문 분석 보고서를 작성하세요.
5. 분석 용도가 단순 내용 파악일 경우(목적 : 읽을 가치가 있는지 판단, 연구 흐름 파악) 다음과 같은 템플릿에 맞춰서 분석 보고서를 작성해주세요.
6. # 논문 요약 보고서, ## 논문 제목, ### 1. 논문의 연구 주제 (이 논문은 무엇을 다루는지에 대해 한 문장으로 요약), ### 2. 논문의 핵심 (논문의 핵심 3가지를 한 문장씩 작성), ### 3. 연구 목적 / 문제정의 (어떤 문제를 해결하려고 하는가?, 기존 연구의 어떤 한계를 지적하는가? 의 내용에 대해 1~3 문장으로 요약) ### 4. 연구 방법 (실험/설문/모델링/시뮬레이션/사례분석 등과 데이터 규모에 대해 정리, 내용의 분량에 따라 1~2문단으로 개조식으로 정리) ### 5. 핵심 결과 (가장 중요한 결과 2~3개, 때에 따라 5개 까지, 중요한 결과 한 개당 1문장으로 정리) ### 6. 차별성/기여점 (기존 연구와 무엇이 다른가? 1~2문장으로 개조식으로 정리) ### 7. 한계점(저자가 인저한 한계, 명확히 보이는 한계)
7. 분석 용도가 세부 내용 분석일 경우 다음과 같은 템플릿에 맞춰서 분석 보고서를 작성해주세요.
8. # 논문 세부 분석 보고서, ## 논문 제목, ### 1. 연구 배경 및 이론적 프레임워크 (어떤 이론에 기반하는가?, 연구 가설은 무엇인가?, 논리 전개가 타당한가?) ### 2. 연구 질문/가설 (명확히 정의되어 있는가?, 조작적 정의가 명확한가?) ### 3. 연구 설계(연구 유형, 변수 구성, 통제 변수 존재 여부, 타당도), ### 4. 데이터 및 표본(표본 크기가 충분한가?, 표본 편향 가능성?, 데이터 수집 방식의 문제는?), ### 5. 분석방법 (통계기법 적절한가? 가정 충족 여부?, 다른 분석이 더 적절하지 않았을까?), ### 6. 결과 해석 (결과가 가설을 충분히 지지하는가?, 과도한 해석은 없는가? 통계적 유의성과 실제적 의미 구분했는가?), ### 7. 한계 및 편향 (저자가 언급하지 않은 한계는?, 재현 가능성?, 잠재적 오류 요인?), ### 8. 학문적 기여(이 분야에서 위치는?, 이론 발전에 기여하는가?, 방법론적 혁신인가?) ### 9. 정책적/실무적 함의 (실제 적용 가능성?, 현실성과 괴리?), ### 10. 후속 연구 아이디어 (확장 가능성?, 다른 맥락에서 적용?, 반박 실험 설계 가능?)
9. 더불어 각각의 분석 보고서 마지막에 "검증" 이라는 항목을 추가해서 방금 답변한 내용 중에서 : 1. 확실히 사실인 것, 2. 불확실한 것, 3. 추측인 것을 구분해서 알려주고, 각 항목에 대해 신뢰도를 상/중/하로 표시해서 정리하세요.
10. 결과를 파일로 저장하세요

[출력 형식]
마크다운 형식, 표 및 그림 사용, 단순 내용 파악은 A4 1~2페이지 분량, 세부 내용 분석은 A4 3~4페이지 분량
톤앤매너 : 개조식으로 간결하고 명확하게 작성하고, 필요시 표와 그림을 적극활용해주세요.
